# Audit 2026-04-28

This short audit captures the current state of the codebase after reviewing the active
admin flow, study flow, validation layer, and the updated documentation.

## Scope checked

- Flask startup and route registration
- study configuration validation
- result payload validation
- admin study editor and study preset loading
- participant study flow with stimulus handling
- dashboard-related status routes
- core project documentation (`README.md`, `docs/01`, `docs/02`)

## Main finding

- The new `finish` card was already required by the frontend flow, but the backend validation
  layer still rejected it as an unknown type and then also expected an answer for it.

## Fix applied

- Added `finish` to the allowed question types in `app/validation.py`.
- Added normalization support for `finish` cards with `title` and `prompt`.
- Excluded `finish` from required answer validation, the same way `participant-id` and
  `stimulus` are excluded.

## Verification done

- Flask app creation with hardware startup disabled completed successfully.
- Route registration smoke check passed.
- Manual validation smoke test for a config containing `participant-id` and `finish` passed.
- Documentation was updated to reflect the current card set, integrations, and saved-study flow.

## Residual risks

- There is still no broad automated test suite for the frontend interaction flows.
- Some optional integration paths depend on external hardware, network reachability,
  or third-party packages and were only checked at code level in this audit.
